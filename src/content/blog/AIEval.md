AI 애플리케이션을 개발하기 전에 먼저 "어떻게 평가할 것인가?"를 정할 필요가 있다. 만들려는 목표가 명확해야 모델 선정과 파인튜닝 방향을 잡을 수 있기 때문이다. 모델을 평가하는 방식은 정말 다양하다 수많은 정량적, 정성적 지표들이 존재하는데, 어떤 평가지표들이 있는지 알아보려고 한다. 추가적으로 현재 진행중인 팀프로젝트에서 AI간의 대화를 통해 가고 싶은 식당을 합의하는 프로세스를 만들기 위해서는 어떤 평가 지표들을 사용해야하는지 선정하려고 한다.

## 1. 도메인 특화 능력
애플리케이션의 핵심 기능에 필요한 능력이 있는지 평가한다. 도메인 특화 능력은 용도별로 다양한 벤치마크가 존재한다.

| 도메인   | 벤치마크              |
| ----- | ----------------- |
| 코딩    | MBPP, SWE-bench   |
| 수학/추론 | MATH, GSM8K       |
| 한국어   | KMMLU, KoBigBench |
| 일반 지식 | MMLU, HellaSwag   |
| 번역    | FLORES, WMT       |
사람이 직접 평가 하면 신뢰도가 높겠지만, 시간과 비용 측면에서의 트레이드 오프가 존재하기 때문에 GPT나 Gemini 같은 대형 LLM 모델을 활용해서 채점하기도 한다. 이를 LLM-as-Judge라고 한다.

분류 문제의 경우에는 정밀도(Precision)나 재현율(Recall), F1 Score를 사용한다.

---
## 2. 생성 능력
LLM이 텍스트를 얼마나 잘 생성하는지 다방면으로 평가한다.

#### 유창성(Fluency)
문장이 문법적으로 올바르고 자연스러운지 평가한다. 자동 지표로는 Perplexity(PPL)가 많이 쓰이고, 점수가 낮을수록 유창하다.

#### 일관성(Coherence)
전체 텍스트가 논리적으로 이어지는지 평가한다. 문장/문단 간 주제가 바뀌거나 앞에서 한 말과 충돌하지 않아야 한다.

#### 충실성(Faithfulness)
주로 번역이나 RAG 시스템에서 중요하게 보는 지표로, 원문이나 소스 문서의 의미를 얼마나 정확하게 반영했는지 평가한다. 자동 지표로는 BLEU, ROUGE, BERTScore 등이 존재한다.

#### 관련성(Relevance)
주로 요약 태스크에서 중요하고, 요약문이 원문의 핵심 내용을 빠뜨리지 않고 잘 다루는지 평가한다. ROUGE 계열 지표를 많이 사용한다.

#### 사실 일관성/사실 비일관성(Factual Consistency/Inconsistency)
사실 일관성은 모델 출력이 사실과 일치하는지를 보고, 사실 비일관성은 모델이 없는 사실을 지어내는 정도를 말한다. 우리가 흔히 듣는 Hallucination을 판단하는 지표이다. FactScore, RAGAS의 faithfulness 등이 사용된다.

#### 안정성(Safety)
욕설, 혐오 표현, 유해 콘텐츠가 생성되지 않는지 평가한다. Perspective API, Llama Guard 같은 전용 모델로 자동화 가능하다.


---
## 3. 지시 수행 능력
사용자의 지시를 얼마나 잘 따르는지 평가하는 항목으로, 에이전트 시스템에서 중요하게 여긴다. 지시 수행 능력은 파인 튜닝 보다 프론프트 엔지니어링으로 개선할 수 있는 부분이 많다.

#### IFEval(Instruction Following Evaluation)
Google이 공개한 벤치마크로 검증 가능한 제약조건("500 단어 이내", "JSON 형식으로 출력") 내에서 자동으로 채점한다.

#### INFOBench
복잡하고 다양한 지시 유형을 커버하는 벤치마크로 여러 조건이 동시에 주어졌을 때 모두 지키는지를 평가합니다.

## 4. 비용과 지연 시간
아무리 성능이 좋아도 실제 서비스가 가능하려면 비용과 시간적인 측면을 무시할 수 없다.

#### 비용(Cost)
입력/출력 토큰 당 비용으로, 트래픽이 많아질수록 모델 선택이 비용에 큰 영향을 미칩니다.

#### TPM(Token Per Minute)
분당 처리할 수 있는 최대 토큰 수. API 제공자가 설정한 rate limit이고, 대규모 배치 처리나 동시 요청이 많을 때 병목이 될 수 있다.

#### TTFT(Time to First Token)
사용자가 요청을 보내고 첫 번째 토큰이 도착하기까지 걸리는 시간으로, 스트리밍 응답에서 체감 반응 속도와 크게 연관이 있다. TTFT가 낮을 수록 사용자가 체감하는 속도가 빠르다.

#### Elo Rating
체스 같은 게임에서 쓰이던 상대적 실력 측정 방식을 LLM 평가에 도입한 방식이다. Chatbot Arena 같은 플랫폼에서 두 모델의 응답을 사람이 비교해서 승/패를 기록한다. 이를 통해 순위를 매기는 상대적 평가

## 합의 프로세스 모델 평가 지표
페르소나 합의 과정에서는 자연스러운 대화가 필요하기에 한국어 벤치마크가 되어있는 모델과 유창성, 안정성, 일관성, IFEval, 비용, TPM, TTFT 를 측정해야한다.
