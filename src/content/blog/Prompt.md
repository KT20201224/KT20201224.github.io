## 프롬프트와 프롬프트 템플릿
프롬프트는 AI에게 주는 하나의 요청 메시지 전체를 의미한다. 
- "이 문제는 어떻게 해결하는게 좋을까?"
- "이 코딩테스트 문제는 무슨 알고리즘을 써서 구현해야 할까?"

즉, 이렇게 AI에게 보내는 메시지 전체를 "프롬프트"라고 정의한다. 그렇다면 **프롬프트 템플릿**이란 무엇일까? 템플릿이란 일정한 모양의 형태로 미리 만들어진 서식을 말한다. 우리가 PPT자료를 만들때 PPT 템플릿을 받아서 우리의 내용을 채웠던 적이 있을 것이다. 이처럼 미리 만들어진 틀(구조, 형식, 규칙)에 맞춰 입력 혹은 출력을 바꿔넣는 형식을 말한다.
```
너는 코딩테스트 요구사항 분석을 첨삭해주는 시니어 개발자야.

입력:
- 문제 제목: {{title}}
- 문제 설명: {{description}}
- 분석 초안:
"""
{{analysis}}
"""

출력 형식:
{
  "score": {...},
  ...
}
```

#### 성능이 좋다는 어떻게 정의할까?
프롬프트의 성능을 평가하기 위해서는 크게 3가지 항목이 존재한다.
1. 정확도 : 질문의 의도를 잘 파악해서, 다른 내용을 말하지 않는가
2. 일관성 : 같은 입력에 대해서 비슷한 출력구조로 답변이 돌아오는가
3. 재현성 : 설계한 프롬프트가 다른 문제/데이터에서도 잘 작동하는가

위 3가지 항목의 완성도를 높이려면 프롬프트도 모델처럼 설계를 거쳐야한다.

#### 좋은 프롬프트의 5가지 요소
###### 1. 역할(Role) 
역할을 부여합니다. "너는 ~ 전문가이다." 정도도 좋지만, 더 나아가서 다음과 같이 최종 사용 시나리오까지 같이 적어주면 좋다. 
"너는 현직에서 일을 하는 시니어 개발자야. 현직 면접자 관점에서, 지원자의 사고과정을 확인한다는 느낌으로 첨삭해줘"

###### 2. 목표(Task)
해야할 일을 명확히 적는 것이 좋습니다. "요약해줘, 정리해줘" 같은 말보다 "분류해줘, 테스트 케이스를 만들어줘, 예외상황을 포함해서 알려줘"처럼 동사 + 산출물을 명확히 적어줘야 합니다.

###### 3. 컨텍스트(Context)
좋은 질문이 곧 좋은 답변을 유도합니다. 필요한 정보만 깔끔하게 정리해서 작성합니다. 내용이 길면 구분을 명확히 해줘야합니다.
```
[문제 제목]
...

[문제 설명]
...

[내가 쓴 요구사항 분석 초안]
...
```
위와 같이 항목을 나누면 모델이 블록단위로 정확하게 인식하기가 쉬워집니다.

###### 4. 형식(Format)
출력형식을 고정합니다. "정리해줘", "작성해줘" 보다 더 자세히 반환 형태를 명시해주면 좋습니다. 예를들어, "마크다운 문법으로 작성해줘", "JSON 형태로 응답해줘"와 같이 상황에 맞는 답변 형태를 유도할 수 있습니다.

###### 5. 제약조건(Constraints)
품질 기준, 응답 길이, 어투, 등의 제약을 걸어줄 수 있습니다. "각 항목에 대하여 300자 이내로 작성해", "면접관처럼 냉정하게", "구현 부분은 제외하고 요구사항만 분석해"처럼 제약 조건을 걸면 내가 원하는 답을 얻기가 쉬워집니다.

#### 고급 기법1 : Few-shot
AI는 "규칙"보다 "패턴"을 더 잘 따라합니다. 위 방법들도 좋지만, 좋은 예시를 1~2개 추가해주면 출력 품질이 훨씬 올라갑니다.
```
[예시 출력]
{
  "score": {...},
  "summary": "...",
  "strengths": ["...", "..."],
  "weaknesses": ["...", "..."],
  "missing_cases": ["...", "..."]
}

[이제 아래 입력에 대해 위와 동일한 형식으로 답해줘]
...
```
#### 고급 기법2 : Step by step으로 시키기
복잡한 작업일수록 여러 단계로 쪼개면 좋은 결과가 나올 수 있습니다. AI로 "요구사항 분석 첨삭"을 진행한다고 가정하겠습니다.
- 1단계 : 
	- "이 분석이 어떤 문제를 설명하는지 한줄로 추론해봐"
	- "문제 번호와 설명이 분석과 일치하는지 판단해"
	- "일치 여부를 `match`/`not_match`로 알려줘"
- 2단계 :
	- "만약 위 단계에서 `match`였다면, 그때 첨삭을 진행해"
